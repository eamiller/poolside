db_layer — Save Data to CouchDB¶

Use CouchDB as a storage backend for notebooks.

class db_layer.Methods(couch_port, database)¶

Update CouchDB with changes from a frontend notebook.

delete_cell(cell_id)¶

Delete a cell from CouchDB.

Warning

Users can delete any cell, whether or not they own it.

save_cell(cell_id, fields)¶

Store notebook cell documents in CouchDB.

Warning

There are NO security measures at the moment.

If a non-empty string or unicode is submitted as the cell_id, it will be accepted as the cell’s ID.

This will be fixed, but it’s a problem right now.

Parameters:
  • cell_id – A cell’s UUID, for both notebook frontends and CouchDB.
  • fields – A dict of key-value pairs to add to this CouchDB document.

If an empty string is provided as the cell_id, a new UUID is generated and a new cell created in the database.

Other “incorrect” inputs will cause an immeidate return.

save_worksheet(worksheet_id, cell_list)¶

Save a worksheet into CouchDB.

Warning

Two open instances of the same worksheet can overwrite each other.

The parameter cell_list is generated on the client side, and there are currently no locks on the CouchDB documents.

Parameters:
  • worksheet_id – A worksheet document’s UUID.
  • cell_list – A list of cell UUIDs contained by this worksheet.
db_layer.new_id()¶

Get a new UUID.

Call this when a notebook cell is created.
